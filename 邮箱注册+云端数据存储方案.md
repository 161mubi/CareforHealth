# 邮箱注册 + 云端数据存储方案

## 概述

将应用从纯前端 localStorage 架构迁移到 Firebase（Auth + Firestore），实现邮箱注册登录并将数据存储在云端，支持跨设备访问。

## 实施步骤清单

- [ ] 1. 创建 Firebase 项目并获取配置（需手动操作）
- [ ] 2. 在 index.html 中引入 Firebase Auth + Firestore CDN
- [ ] 3. 新增登录/注册视图（邮箱+密码表单、忘记密码）
- [ ] 4. 新建 firebase-config.js：初始化 Firebase、认证逻辑、状态监听
- [ ] 5. 重构数据层：localStorage 函数改为 Firestore 异步读写
- [ ] 6. 移除旧的 URL 参数多用户机制，改用 Firebase Auth 身份识别
- [ ] 7. 实现本地旧数据一次性迁移到云端的提示与逻辑
- [ ] 8. 配置 Firestore 安全规则（用户只能访问自己的数据）

---

## 为什么选 Firebase

- Google 提供的免费云服务，无需自建服务器
- **免费额度**：Auth 无限用户、Firestore 每天 5万次读 / 2万次写 / 1GB 存储
- 对于个人健康记录应用完全够用
- 纯前端 SDK，无需额外后端代码

## 架构变化

```
当前架构:  浏览器 → localStorage（数据存在本地，换设备丢失）
新架构:    浏览器 → Firebase Auth（身份认证）+ Cloud Firestore（云端数据库）
```

## 准备工作（手动操作）

1. 前往 [Firebase Console](https://console.firebase.google.com/) 创建项目
2. 启用 **Authentication > Email/Password** 登录方式
3. 创建 **Cloud Firestore** 数据库
4. 在项目设置中获取 Firebase 配置（apiKey, projectId 等）

## 代码改动详情

### 1. 引入 Firebase SDK

在 `app/index.html` 的 `</body>` 前添加 Firebase CDN 脚本（Auth + Firestore），无需 npm/构建工具。

### 2. 新增登录/注册页面

在 `app/index.html` 中添加一个 `view-auth` 视图，包含：

- 邮箱 + 密码输入框
- "登录" / "注册" 切换按钮
- "忘记密码" 链接（调用 Firebase `sendPasswordResetEmail`）
- 错误提示区域

未登录时仅显示此视图，登录后显示主应用。

### 3. 新增 Firebase 初始化与认证逻辑

新建 `app/firebase-config.js`，包含：

- Firebase 初始化配置
- `onAuthStateChanged` 监听登录状态
- 登录 (`signInWithEmailAndPassword`)、注册 (`createUserWithEmailAndPassword`)、登出 (`signOut`)、重置密码函数

### 4. 数据层从 localStorage 迁移到 Firestore

在 `app/app.js` 中重构以下 6 个核心数据函数（共约 20 处 `localStorage` 调用）：

- `getData()` / `saveData()` — 偏头痛和经期记录
- `getCustomOptions()` / `saveCustomOption()` / `removeCustomOption()` — 自定义选项
- `getRemovedDefaults()` / `addRemovedDefault()` — 已删除的默认选项

**Firestore 数据结构**：

```
users/{uid}/records    — 存储偏头痛/经期记录（每条记录一个文档）
users/{uid}/settings   — 存储用户偏好（自定义选项、已删除默认项）
```

数据读写改为异步（`async/await`），页面初次加载时从 Firestore 拉取数据缓存到内存变量，后续操作同时更新内存和 Firestore。

### 5. 移除旧的多用户机制

当前基于 URL 参数 + localStorage 前缀的多用户方案将被 Firebase Auth 取代：

- 删除 `getUserPrefix`、`initUserKeys`、`getAllUsers`、`switchUser`、`showUserPicker` 等函数
- 删除用户选择模态框 (`modal-user`)
- 标题改为显示用户邮箱或昵称
- 新增"退出登录"按钮

### 6. 数据迁移支持

首次登录时检测 localStorage 中是否有旧数据，若有则提示用户"是否将本地数据导入云端"，一次性写入 Firestore 后清除本地数据。

### 7. Firestore 安全规则

设置规则确保每个用户只能读写自己的数据：

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId}/{document=**} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
  }
}
```

## 需要注意的问题

- 数据读写变为异步，所有依赖数据的 UI 渲染需要适配
- 首次加载需要网络连接；可考虑启用 Firestore 离线持久化作为缓存
- Firebase 配置中的 `apiKey` 是公开的（这是正常的，安全由 Firestore Rules 保障）
